# SKILL.md 编写

## 摘要

Skill 的本质是把“个人经验”固化为“可重复执行流程”，让团队交付质量更稳定。

## 详细内容

### 业务背景

单靠口头约定很难长期维持工程质量。Skill 文档把检查步骤、文档要求、交付标准固化下来，降低人员变动带来的质量波动。

### 一个有效 Skill 应包含

- 触发条件：什么情况下必须使用。
- 目标产物：需要生成哪些文件、跑哪些命令。
- 验证标准：什么叫完成，失败如何回退。
- 边界约束：哪些行为禁止（例如跳过测试、漏写日志）。

### 在本仓库的价值

- `build-check`：保证每次改动都经过统一门禁。
- `dev-logs`：让每次开发有可追踪记录。
- `repo-structure-sync`：防止架构文档和实际目录脱节。

### 风险与边界

- Skill 太宽泛会执行漂移，太细碎会增加执行负担。
- 如果不绑定验证命令，Skill 容易变成“文档装饰品”。
