# 自动测试生成

## 摘要

测试不是补作业，而是交付保护网。AI 参与开发时，测试是防止“看起来没问题但上线出问题”的关键保障。

## 详细内容

### 业务背景

在快速迭代中，需求和代码同时变化。若没有自动测试，回归成本会随着功能数量线性上升，最终拖慢发布节奏。

### 在本项目的落地方式

- 数据层变更后，运行 `npm run test` 保证知识树结构完整。
- 同步脚本变更后，运行 `check_errors` 保证 TypeScript、Lint、Build 全通过。
- 新增同步链路时，补“命令级验证”（脚本是否可重复执行、是否幂等）。

### 指标建议

- 构建通过率（Build Pass Rate）
- 回归缺陷数（Regression Count）
- 单次发布准备时间（Release Ready Time）

### 风险与边界

- 只测 happy path 会掩盖真实风险，尤其是数据同步脚本。
- 测试与业务规则脱节时，会出现“测试通过但业务失败”。
